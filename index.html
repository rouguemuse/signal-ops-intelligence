Metrics Dictionary

          
Prime Cost Percentage

          
Prime Cost % = ((Cost of Goods Sold (COGS) + Total Labor Cost) / Gross Revenue) * 100

          
Indicates combined raw food costs and payroll margins. Target standard is 55-60%.

          
          
Sales Per Labor Hour (SPLH)

          
SPLH = Net Sales / Total Labor Hours Worked

          
Standardized measurement of labor productivity. Free from wage rate deviations. Target FSR: $75-$110.


          
Off-Peak Labor Leakage

          
Identifies slow periods where sales fall below the core operational threshold and hourly labor percentage exceeds 45%.

Severity & Confidence Scoring Rubrics

          
Anomalies are scored based on two separate vectors to preserve rating clarity:

          
Severity Score (3-15) = Financial Impact (1-5) + Frequency (1-5) + Operational Risk (1-5)

          
Confidence Score (1-5) = (Data Completeness (1-5) + Metric Stability (1-5) + Input Quality (1-5)) / 3

        


        
        

          
Diagnostic Methodology

          
Out-of-Bounds Off-Peak Labor Leakage

          
Identifies mid-day sales gaps where payroll cost percentage exceeds 45% and sales drop below core staffing baselines ($150). Opened/Closed setup shifts are excluded.

          
          
Server Void Outlier (Z-Score Deviation)

          
Uses statistical standard deviations to evaluate individual employee void rates against peer averages on the same shift types. Flagged if Z-Score ≥ 1.5, protecting the heuristic against global shift errors.

        


        
        

          
Privacy & Security Model

          
Zero Server-Side Storage: Ingested files are parsed within local browser memory using HTML5 FileReader. Restaurant transaction numbers, employee payroll profiles, and operational margins never leave the computer.

          
Reduced Exposure: Client-side computing mitigates compliance exposure (PCI, salary details) and external security risk.

        


      

    



    
    

      

        
Why This Project Matters for AI Operations

        
Signal demonstrates the same core competencies required in AI evaluation, data operations, and LLM rating frameworks:

        
        
1. Structuring Messy Unstructured Inputs

        
In AI systems, prompt outputs and document files are highly unstructured and drift frequently. Signal's column parser in parsers.js uses fuzzy substring heuristics to clean and map arbitrary external tables to unified schema keys—similar to normalizing LLM JSON output structures.


        
2. Validation & Anomaly Guardrails

        
Evaluators must catch malformed or invalid runs before calculations start. Signal pre-validates files, intercepts negative hours, interpolates missing ingredients cost with averages, and logs telemetry warnings—matching LLM verification and data monitoring practices.


        
3. Grading and Scoring Frameworks

        
Assessing model accuracy, toxicity, or safety requires strict quantitative scoring. Signal translates business targets (SPLH ratios, margin cutoffs) into repeatable mathematical criteria, and scores anomalies using a multi-dimensional rubric:

        
Severity Score = Financial Impact + Frequency + Operational Risk + Confidence Level

        
This is identical to rating LLM errors based on severity, occurrence probability, user safety, and evaluator model certainty.


        
4. Explainability & Observability

        
In evaluation pipelines, grading labels must be transparent and actionable. Signal does not just flag anomalies; it outputs the score components, explains the root cause, and generates prioritized, step-by-step remediation plans. This models the explainable AI (XAI) design required in production systems.
